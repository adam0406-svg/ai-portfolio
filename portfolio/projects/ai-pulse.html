The Problem

AI is moving fast. Staying informed shouldn't require a part-time job.

The AI news cycle is overwhelming. Between Reddit threads, Hacker News discussions, research paper announcements, product launches, and industry commentary, there are hundreds of potentially relevant signals per week. Most of them are noise. A few are genuinely important. Without a system, you either consume too much and still miss what matters, or you stop trying and fall behind.

For people working in AI transformation, consulting, or product roles, staying current isn't optional — it's part of credibility. But manually curating and reading across 15+ sources, filtering for relevance, summarising, and identifying what's signal versus marketing copy takes 3–5 hours per week that most people simply can't allocate.

The problem isn't that information doesn't exist — it's that there's too much of it, poorly organised, distributed across incompatible formats, with no layer to distinguish genuine development from hype.

The Solution

Three agents. Multiple sources. One structured digest, delivered on demand.

AI Pulse is a multi-agent intelligence pipeline built on FastAPI and the Anthropic API. It monitors Reddit (AI-related subreddits), Hacker News, and a configurable set of RSS feeds in parallel, then runs the collected content through a three-stage agent chain: Curator, Briefer, and Buzzword Detector.

The Curator agent filters and ranks collected content for relevance, recency, and signal quality — removing noise before it reaches downstream agents. The Briefer agent synthesises the curated set into a structured narrative digest with clear categories (model releases, tooling, research, industry moves). The Buzzword agent reads the final digest and flags terms being overused or misused — tagging hype language so the reader knows what to treat critically.

The pipeline runs with parallel async fetching from multiple sources simultaneously, significantly reducing total processing time. Output is a clean, categorised weekly brief rendered in the UI — readable in under 10 minutes and usable as the basis for team briefings, newsletter content, or just staying sharp.

Technical Approach

Direct API integration with parallel async architecture.

AI Pulse demonstrates direct Anthropic API usage — no framework abstraction, raw API calls with structured prompting. This intentional choice shows understanding of the underlying model layer that frameworks sit on top of. The multi-agent structure is implemented at the application level rather than via an orchestration framework.

Stack & Architecture
Backend
FastAPI (Python) with async/await throughout — sources are fetched concurrently, not sequentially, using asyncio. Significantly faster than synchronous implementations when hitting multiple APIs.
Sources
Reddit (PRAW), Hacker News API, and RSS feeds via custom source connectors in the sources/ module. Each source is independently configurable and fault-tolerant — one failing source doesn't kill the pipeline.
Agents
Three agents via direct Anthropic API: Curator (filter/rank), Briefer (synthesise/structure), Buzzword (detect/flag). Each agent receives structured context and returns structured JSON. The pipeline is linear: each agent's output feeds the next.
Output
Structured JSON digest rendered in a single-file frontend (index.html). No build step. The UI parses and displays sections: top stories, category breakdown, buzzword report, and signal/noise rating per item.
Lessons & Pitfalls

What this project exposed about real-world pipeline complexity.

!

Source quality variance is the hardest problem

On a slow news week, the Curator agent receives lower-quality input and the digest reflects that — there's no way to synthesise high-quality output from thin source material. The pipeline is only as good as what it ingests on a given day. This is a fundamental constraint of real-time content aggregation, not a fixable bug.

!

Rate limits require more defensive architecture than expected

Reddit's API rate limits and Hacker News's response time variance caused intermittent failures during parallel fetching. The fix was per-source retry logic with exponential backoff — but the lesson was that any production pipeline hitting external APIs needs rate limit handling as a first-class concern, not an afterthought.

i

The buzzword detection agent adds surprisingly high value

Initially added as a minor feature, the Buzzword agent turned out to be one of the most useful outputs. Knowing that "autonomous agents" appears 47 times in a week's digest with low signal correlation is more useful than the count alone. The meta-analysis layer — commenting on the information, not just synthesising it — is a pattern worth applying more broadly.

What's Next

Making this useful at the team and organisational level.

  • Personalisation by role — a product manager gets a different digest emphasis than a data scientist or an executive
  • Slack / email delivery — push the weekly digest rather than requiring a pull, with configurable cadence
  • Organisation-specific keyword tracking — monitor for named competitors, specific technologies, or regulatory developments relevant to a client
  • Historical trending — compare this week's digest against prior weeks to surface emerging vs. sustained trends
  • Source quality scoring — rank sources dynamically based on how often their content survives Curator filtering
