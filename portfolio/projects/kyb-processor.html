The Problem

KYB compliance is a bottleneck in every financial services onboarding process.

Know Your Business (KYB) checks are a regulatory requirement for any financial services business, fintech, or platform onboarding corporate clients. The process involves gathering basic business information, verifying registration and legal structure, checking beneficial ownership, screening against sanctions and PEP lists, and evaluating overall risk.

Done manually, KYB is slow, inconsistent, and resource-intensive. Compliance analysts query registries manually, copy data between systems, apply judgment calls that vary by analyst, and produce reports in inconsistent formats. For high-volume onboarding — as experienced by any growing B2B platform — the manual approach doesn't scale.

The regulatory requirement doesn't go away. The question is whether it's handled by humans doing repetitive data work, or by an automated system that handles the routine and escalates the exceptions.

This project demonstrates how a visual workflow automation tool (n8n) combined with an AI API integration can compress the routine parts of KYB to near-zero human time, while keeping a human in the loop for edge cases and escalations.

The Solution

An event-driven n8n workflow that handles data collection, AI assessment, and reporting automatically.

The KYB Processor is built as a visual workflow in n8n — a node-based automation platform that connects services and APIs without requiring a traditional codebase. The workflow is triggered by a form submission or webhook event, kicks off an automated sequence, and produces a structured KYB verdict with an audit trail.

The workflow pulls business data from available public registries and structures it into a standardised profile. It then passes this profile to an AI API integration with a prompt engineered to evaluate the business against defined KYB criteria — ownership transparency, jurisdictional risk, sanction exposure, operational legitimacy — and produce a structured verdict: Pass, Review Required, or Fail.

Each run produces a timestamped record with the data sources consulted, the AI's assessment reasoning, and the final verdict. Failed or Review-Required cases are routed to a human analyst. Passed cases are logged and the onboarding process proceeds automatically. The workflow is fully configurable — criteria, thresholds, and routing rules can be adjusted without modifying code, purely through n8n's visual interface.

Business Value

Why this matters for any business with a compliance function.

Auto
Routine cases handled without analyst involvement — human time reserved for exceptions only
Consistent
Every business checked against the same criteria, every time — no analyst-to-analyst variation in risk judgment
Auditable
Timestamped records with data sources and reasoning — built-in compliance documentation for every check

This pattern — automated first-pass with AI, human review for exceptions — is the correct architecture for most compliance workflows. It treats human analyst time as the expensive, scarce resource it is, and applies it where judgment is genuinely required rather than where data lookup is just slow.

The use of n8n rather than a custom codebase is itself a design decision with business implications: non-technical compliance staff can read, adjust, and audit the workflow without involving engineering. When regulations change (and they do), the workflow can be updated visually in hours rather than weeks of development cycles.

Technical Approach

Why n8n, and what it demonstrates about AI integration patterns.

This project was specifically chosen to demonstrate workflow automation as a distinct AI integration pattern — separate from writing code with an AI framework or calling the API directly. n8n represents a class of tools that are highly relevant in enterprise contexts: visual, accessible to non-engineers, and capable of connecting the full ecosystem of business tools.

Stack & Architecture
Platform
n8n — open-source node-based workflow automation. The entire workflow logic, data flow, and conditional routing is configured visually. No backend codebase required. Runs self-hosted or cloud.
Trigger
Webhook node receives the form submission payload. Can be replaced by any trigger — Zapier-style form tools, CRM events, email parsing, or direct API calls from an onboarding portal.
Data
HTTP Request nodes query public business registries and structured public data sources. Data transformation nodes normalise the responses into a consistent format before AI assessment.
AI API
HTTP Request node calling the AI API (Anthropic or equivalent). The prompt includes the structured business profile, the KYB evaluation criteria, and instructions to return a structured JSON verdict with reasoning.
Routing
IF/Switch nodes route outcomes: Pass cases to automated logging; Review/Fail cases to a human analyst queue (email notification, Slack message, or CRM task creation). The routing logic is visually configurable.
Audit
Every run writes a structured record to a database or Google Sheet: timestamp, input data, sources consulted, AI assessment, routing decision. Full audit trail without additional tooling.
Lessons & Pitfalls

What workflow automation taught me about where the real complexity lives.

!

Public registry data is inconsistently available across jurisdictions

Business registry APIs vary dramatically in quality, accessibility, and data completeness by country. UK Companies House is well-structured and free; many EU registries require paid access or manual lookup; some jurisdictions have no accessible API at all. Any production KYB system needs a strategy for jurisdictional coverage gaps — the workflow currently handles this with fallback logic that flags incomplete data rather than failing silently.

!

AI verdict confidence needs to be surfaced, not hidden

Initial versions returned a binary Pass/Fail verdict. This was operationally problematic: the AI's confidence varied considerably based on data quality, but the output looked identical. Adding a confidence score and a data completeness indicator — flagging cases where the AI was working with thin data — dramatically improved the utility of the Review Required category and reduced analyst frustration with false escalations.

i

The visual workflow is a compliance communication tool, not just automation

When the workflow was shown to a compliance officer (not a developer), they could immediately understand what was happening at each step, where data came from, and where the human review points were. This transparency is an advantage of visual workflow tools in regulated contexts — the workflow itself becomes part of the audit documentation, not just a black box that produces outputs.

What's Next

Extending this into a full regulatory workflow component.

  • Full beneficial ownership mapping — traverse ownership chains to identify ultimate beneficial owners above the regulatory threshold (typically 25%)
  • Document verification — accept uploaded documents (articles of incorporation, certificates of good standing) and use AI to verify contents match registry data
  • Ongoing monitoring — re-run KYB checks on existing clients when registry data changes or a sanctions event occurs, not just at onboarding
  • Risk scoring dashboard — aggregate verdict data across all processed businesses to identify portfolio-level risk concentration
  • Multi-jurisdiction expansion — add structured handling for priority markets beyond the current coverage, with jurisdiction-specific evaluation criteria
  • CRM integration — write KYB results directly to Salesforce or HubSpot records, linking compliance status to the commercial relationship
